Turtle Crossing Game

This is what I do.

Overview

This Turtle Crossing Game project was implemented using Python and the Turtle module. The game features a player-controlled turtle that moves up the screen and attempts to avoid moving cars that randomly appear. The goal is to reach the finish line at the top of the screen, where the level will increase, and the speed of the cars will increase as well. If the player collides with a car, the game ends. The project includes a scoring system that tracks the player's level, and the game resets when a collision occurs.

Explanation of the Code

main.py

  • Turtle Screen Setup: The screen is created with specific dimensions of 600x600 pixels and uses the Screen class from the turtle module. The screen updates every 0.1 seconds using the screen.tracer(0) method.
  • Game Loop: The main game loop runs continuously, updating the screen and controlling the movement of the cars and the player. The player can control the turtle using the 'Up' arrow key to move it upwards.
  • Collision Detection: If a car's distance from the player is less than 20 pixels, the game ends, and a "Game Over" message is displayed.
  • Leveling Up: When the player reaches the finish line, their score is incremented, and the player’s position is reset to the starting position. At the same time, the car speed is increased to make the game more challenging.

car_manager.py

  • Car Creation: The CarManager class is responsible for creating the cars. It uses a random number generator to decide when a car should appear and sets random positions for the cars along the Y-axis.
  • Car Movement: Each car moves towards the left of the screen at a constant speed, and their movement is controlled by the car_move method.
  • Increasing Car Speed: As the player progresses, the speed of the cars increases by a defined amount with each level.

player.py

  • Player Setup: The player is represented by a turtle with a starting position at (0, -280). The turtle moves upwards when the player presses the 'Up' key.
  • Finish Line Detection: The player’s goal is to reach the top of the screen, at which point the is_at_finish method returns True, signaling the player has reached the finish line.
  • Resetting Player Position: If the player reaches the finish line, they are moved back to the starting position.

scoreboard.py

  • Scoreboard Setup: The Scoreboard class tracks the player's level, which is displayed at the top of the screen. The score updates when the player reaches the finish line.
  • Increasing Score: The score increases by 1 each time the player reaches the finish line.
  • Game Over: When the player collides with a car, the game ends, and the "GAME OVER" message is displayed at the center of the screen.

data.txt

  • High Score Storage: This text file stores the highest score achieved in the game. It’s read and written by the Scoreboard class to persist the high score between game sessions.

Gameplay Mechanics

  • Game Setup: The game initializes with a screen size of 600x600, with the player (turtle) represented by a turtle object and the cars represented by square shapes. The player's goal is to avoid cars and reach the finish line to level up.
  • User Input: The player controls the turtle using the 'Up' arrow key. The turtle moves upwards with each press of the 'Up' key.
  • Car Movement and Speed: Cars are generated at random Y-positions and move from right to left. As the player progresses, the speed of the cars increases.
  • Game Over: The game ends if the turtle collides with a car, and the player is shown a "GAME OVER" message.

How I Developed the Game

I gained hands-on experience with Python through the 100 Days of Code: The Complete Python Pro Bootcamp for 2023. I implemented this Turtle Crossing Game as a practical project to understand object-oriented programming (OOP) concepts. I created classes for the player, cars, and scoreboard, which made the game logic modular and easier to manage.

This project helped me break down the game into smaller, manageable parts. The Player class interacts with the CarManager to avoid collisions, while the Scoreboard tracks the player’s progress and updates the score. Encapsulation was key, as each class handles its own functionality, such as controlling the player’s movement or adjusting the cars’ speed.

I used the Turtle graphics module to visually represent the player, cars, and game elements. Additionally, I utilized the random module to generate cars at different Y-positions and control their speed as the game progressed, ensuring a dynamic and engaging player experience.

Conclusion

This Turtle Crossing Game is a fun project that helped me practice core programming concepts in Python, including classes, functions, and handling user input. The game has simple yet engaging mechanics, and the increasing difficulty makes it challenging to play. I used the Turtle module to create an interactive and visual experience, and I enjoyed implementing file handling to track the high score.
