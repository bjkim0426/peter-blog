Snake Game

This is what I do.

Overview

This Snake Game project was implemented using Python and the Turtle module. The game features a snake that moves around the screen, collects food, and keeps track of the score. The game ends if the snake runs into itself or the boundaries of the screen. The project includes a scoring system and keeps track of the highest score between sessions.

Explanation of the Code

main.py

  • Turtle Screen Setup: The screen is created with specific dimensions of 600x600 pixels and a black background.
  • Game Loop: The main game loop runs continuously, updating the screen and controlling the snake's movement. It listens for user inputs (arrow keys) to change the snake's direction.
  • Snake Movement: The snake moves based on the direction set by the user. It moves segment by segment, with the head of the snake leading the movement.
  • Collision Detection: The game checks for collisions with food and screen boundaries. When the snake eats food, its body grows, and the score increases. The game resets if the snake hits the boundary or itself.
  • Game Termination: The game stops if the snake collides with itself, and the score is reset.

food.py

  • Food Creation: The food is created as a circle (using the Turtle module) and placed at a random position on the screen.
  • Food Color: The food color is randomized every time the food appears on the screen.
  • Food Refresh: After the snake eats the food, the food moves to a new random position on the screen.

scoreboard.py

  • Score Tracking: The score is tracked using the score variable, which increases each time the snake eats food.
  • High Score: The high score is stored in a file called data.txt. If the current score exceeds the high score, the new score is saved.
  • Score Display: The score and high score are displayed at the top of the screen in yellow text.

snake.py

  • Snake Creation: The snake is created as a series of segments, each represented by a Turtle object. The snake starts with 3 segments.
  • Snake Movement: The snake moves by updating the position of each segment. The head of the snake moves first, and the other segments follow.
  • Snake Extension: Every time the snake eats food, a new segment is added to its body.
  • Direction Control: The snake's direction is controlled by the arrow keys (Up, Down, Left, Right). The direction is updated using the setheading method.
  • Collision with Self: The game checks if the head of the snake collides with any of its segments, ending the game if this happens.

data.txt

  • High Score Storage: This text file stores the highest score achieved in the game. It‚Äôs read and written by the Scoreboard class to persist the high score between game sessions.

Gameplay Mechanics

  • Game Setup: The game initializes with a screen size of 600x600, with the snake represented by a series of white squares. The food is a colored circle, and the scoreboard is displayed at the top.
  • User Input: The player controls the snake using the arrow keys. The snake moves in the direction corresponding to the pressed arrow key.
  • Food Consumption: When the snake's head collides with the food, the food changes location and the snake grows by adding a new segment to its body. The score increases by 1 point.
  • Game Over: The game ends if the snake collides with the screen border or its own body. The score is reset, but the high score is saved.

How I Developed the Game

I learned the basics of Python through the 100 Days of Code: The Complete Python Pro Bootcamp for 2023. I implemented this Snake Game as a practical project to understand object-oriented programming (OOP) in Python, event handling, and the Turtle module.

The project helped me learn to break down a problem into smaller, manageable parts. I created the snake, food, and scoreboard as separate classes, making it easier to manage and update the game state.

I incorporated file handling by saving the high score in a text file (data.txt), which helped me understand how to read and write data persistently.

I used the time.sleep method to control the speed of the snake‚Äôs movement and to make the game playable at a reasonable pace.

Conclusion

This Snake Game was a great way to apply what I learned in Python. The project reinforced concepts like classes, functions, loops, and conditionals. It also introduced me to working with the Turtle graphics library, which was a fun and visual way to explore programming logic. By breaking down the project into smaller components, I was able to successfully develop the game and understand key programming concepts.
